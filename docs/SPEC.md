# 安卓本地录音 → 腾讯云 COS 上传工具（第一期）— 验收与边界

**状态**：草案（自用 MVP）  
**最后更新**：2026-04-11  
**语言**：中文 UI only（不引入 i18n）  
**云存储**：第一期仅 **腾讯云 COS**（不抽象多云）

---

## 1. 产品一句话

在 **Android 14+（minSdk 34）** 上，扫描**系统录音机**落到媒体库的录音文件，展示列表与**同步状态**；用户填写 COS 凭证与桶信息后，在**有网络且系统允许后台**时尽量快传，若受阻则**当日（设备本地时区自然日内）**完成上传。

---

## 2. 非目标（Non-goals）— 第一期明确不做

以下不做，避免范围膨胀与“用 AI 做出来很散”：

- **Agent / MCP / 服务端代理**：不提供对外的 Agent 消费接口；不在本期做转写、向量检索、摘要。
- **多云与可插拔存储**：仅腾讯云 COS；不出现 S3/阿里云等配置入口。
- **完整安全体系**：不承诺端到端加密、不承诺合规认证；仅遵守文档第 6 节「最低密钥卫生」。
- **账号体系与多用户**：无登录注册、无团队权限模型。
- **实时「录完即传」硬 SLA**：不承诺秒级；仅承诺「尽快 + 当日兜底」（见验收用例）。
- **保证覆盖所有 OEM 私有目录**：以 `MediaStore` 可见的录音机产物为主；不承诺抓取未进入媒体库的沙盒文件。

---

## 3. 页面与信息架构（第一期）

1. **录音列表页（主页）**  
   - 列表项至少展示：**标题、时长、修改时间、文件大小、同步状态**。  
   - 状态枚举固定为：`未同步` / `上传中` / `已同步` / `失败（可重试）` / `已暂停`（若实现暂停策略）。  
   - 提供：**下拉刷新**、单条 **立即上传/重试**（若技术允许，再提供批量「全部重试」）。

2. **COS 配置页**  
   - 字段：`SecretId`、`SecretKey`、`地域 region`、`存储桶 bucket`、`对象前缀 prefix`（可选但建议有默认值如 `recordings/`）。  
   - 提供：**保存**、**测试连接**（至少验证凭证与桶可访问；推荐试传极小对象或 `HeadBucket` 等价能力）。  
   - 所有说明与错误提示：**中文**。

---

## 4. 同步与调度原则（实现约束，供验收引用）

- **主键**：以 `MediaStore` 的稳定标识（如 `Audio.Media._ID`）+ `content://` URI 作为本地主键；上传成功后持久化 `object_key` 与（可选）`etag`/`size`。  
- **写入稳定**：上传前需避免「文件仍在写入」；采用短窗口检测（如连续若干秒大小不变）再入队或再开始传。  
- **队列**：使用 **WorkManager**（或等价可靠队列）；进程被杀、重启后队列不丢（以本地 DB 为准）。  
- **重试**：失败指数退避；**用户打开 App 时**必须触发一次队列巡检/补传。  
- **当日兜底**：以**设备本地时区**的自然日界定「今天」；到达当天较晚时段（实现可自行选例如 22:00 后）应提高重试频率或在通知/列表中明确提示「仍在今日目标内」。

---

## 5. Definition of Done（DoD）— 合并 / 交付前必须满足

- minSdk **34**；目标 SDK 符合当前上架惯例（由实现者选定，但需在 README 写明）。  
- 列表页能稳定展示录音机来源音频（基于 `MediaStore`），且同步状态与本地 DB 一致。  
- COS 配置可保存、可测试；保存后上传使用 **COS 官方/推荐 SDK** 路径（实现细节不限，但需可维护）。  
- **密钥卫生**：`SecretKey` 不打日志、不进 Git；配置仅存应用私有存储（或等价安全存储）。  
- 断网、杀进程、重启三类场景下，**不出现静默丢队列**（除非用户清空应用数据——此时行为需在 UI/文档诚实说明）。  
- 中文 UI 完整覆盖两页主要路径与常见错误提示。  
- 提供简短 `README`：**权限说明**、**已知限制**（Non-goals）、**如何填 COS**。

---

## 6. 最低密钥卫生（不算「安全体系」，但必须做到）

- 使用 **CAM 子账号密钥** 优于主账号密钥；权限收敛到目标桶最小读写集合（实现者按腾讯云最佳实践配置）。  
- 不在截图、日志、崩溃上报中附带密钥。  
- 提示用户：手机丢失且未锁屏时，桶凭证可能泄露——属于可接受风险需在首次保存时一句话告知。

## 7. 验收用例（10 条）— 可逐条打勾

每条通过标准：**行为符合描述 + UI 状态正确 + 无崩溃**。

1. **首次安装授权**  
   - 冷启动后，在未授予媒体读取权限时，列表页给出中文引导；授权后能加载出系统录音机产生的至少一条历史录音（若设备上存在）。

2. **配置 COS 并测试**  
   - 在配置页填入正确参数后，「测试连接」成功；填入错误 `SecretKey` 时，显示明确中文错误（非空白/非仅 Toast 一闪而过即可，具体形态实现自定但需可读）。

3. **未同步 → 已同步（Wi‑Fi 正常）**  
   - 选择一条 `未同步` 录音，触发上传；在稳定网络下进入 `上传中` 并最终 `已同步`；云端对象路径符合 `prefix + 合理文件名规则`。

4. **断网排队**  
   - 开启飞行模式或关闭网络后触发上传：状态进入可理解的「待上传/未同步/失败」之一，但**文案不得暗示数据丢失**；恢复网络后**无需用户手动**也能在合理时间内自动继续并最终 `已同步`（允许 WorkManager 延迟）。

5. **大文件续传/中断恢复（若实现分片则测分片；若 MVP 仅整体重传则测重传）**  
   - 在上传过程中强制断网或杀进程：恢复后同一条录音不会重复产生多个无关联云端副本，或至少以固定 `object_key` 幂等覆盖（实现需在 README 说明策略）。

6. **杀进程不丢队列**  
   - 上传中途划掉任务栈或强制停止（若系统允许测试）：重启 App 后，该条仍为可继续状态，并最终可达 `已同步`。

7. **当日兜底（简化验收）**  
   - 模拟整日网络不可用直至晚间某时刻再恢复：恢复后当日内能完成上传；若跨自然日，列表需诚实显示为未同步/失败，并允许一键重试（不要求「跨日自动算昨天」）。

8. **重复扫描不产生重复 DB 记录**  
   - 多次下拉刷新或重启 App：同一条 `MediaStore` 录音在本地 DB 仅对应一条记录，状态不乱跳。

9. **录音写入中不误传**  
   - 若可构造「刚生成且仍在增长」的文件：不应在大小仍在变化时标记 `已同步`；稳定后才进入上传流程。

10. **清应用数据后的诚实行为**  
    - 清除存储后：COS 配置与本地队列丢失属预期；UI 中文提示用户；云端已存在对象不在本期强制自动对齐（可作为已知限制写在 README）。

## 8. 与后续「Agent 消费」的耦合点（本期仅预留，不实现）

本期建议在 DB 中预留字段（可为空）：**全局唯一 `recording_id`（UUID）**、**`object_key`**、**`content_sha256`（可选）**。  
Agent 相关能力另立文档与验收标准，不在本期 DoD 内。
